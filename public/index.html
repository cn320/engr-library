  1. สรุปการเก็บ Requirement ระบบยืม-คืนหนังสือ ครั้งที่ 1 (17 มกราคม 2556)

    1. ลูกค้าต้องการให้สร้างระบบยืม-คืนได้สะดวก, รวดเร็วกว่าระบบเดิมที่ใช้แผ่นกระดาษจดเก็บสถิติการยืม-คืน
    2. ลูกค้าต้องการให้สร้างระบบที่เป็นฐานข้อมูลยืม-คืน เพื่อเก็บสถิติของผู้ยืมที่แก้ปัญหาเนื้อที่กระดาษที่เก็บข้อมูลยืม-คืนของแต่ละบุคคลไม่พอเขียน และปัญหาลายมือที่อ่านไม่ออกของผู้ยืมเวลากรอกรายละเอียดการยืม
    3. ลูกค้าต้องการให้ระบบที่สร้างมีแจ้งเตือนค่าปรับของบุคคลที่เลยกำหนดการคืนหนังสือแบบอัตโนมัติ และคำนวณค่าปรับให้เสร็จ
    4. ลูกค้าต้องการให้สร้างฐานข้อมูลที่เก็บข้อมูลหนังสือทั้งหมดที่มีอยู่ใน Resource Center
    5. ลูกค้าต้องการให้มีบุคคลากรมาดูแลระบบอยู่ตลอดหลังจากเปิดใช้งาน

  2. Application ที่จะพัฒนาและผู้มีส่วนได้ส่วนเสีย

    1. สร้างระบบยืม-คืนหนังสือในลักษณะการกรอกข้อมูลโดยจะมี
      1.1 ระบบ Login ของผู้เจ้าหน้าเพื่อไปสู่หน้าการยืม-คืน
      1.2 ระบบข้อมูลแสดงรายละเอียดของหนังสือที่ผู้ยืมคนนั้นยืมออกมาเป็น Table
      1.3 ระบบการเพิ่มหนังสือใหม่ผ่านการกรอกข้อมูล
      1.4 ระบบลบหนังสือเก่าผ่านการ Search และกรอกข้อมูล
      1.5 ระบบการ Search หนังสือที่ต้องการ
      1.6 ระบบการตั้งค่าที่เจ้าหน้าที่สามารถกำหนดจำนวนวันยืม และอัตราค่าปรับ
      1.7 ระบบ Database ที่เก็บจำนวนหนังสือทั้งหมด
      1.8 ระบบ Database ที่เก็บสถิติการยืม ณ ปัจจุบัน
      1.9 ระบบการแจ้งเตือนค่าปรับที่ยืมเกินวันที่กำหนดให้กับเจ้าหน้าที่
    2. ผู้มีส่วนได้
      2.1. เจ้าหน้าที่ห้องสมุดอำนวยความสะดวกในกระบวนการยืม-คืนหนังสือได้รวดเร็วขึ้น และตรวจสอบข้อมูลของผู้ยืมได้ง่ายยิ่งขึ้น
      2.2. นักศึกษา และบุคคลที่ต้องการยืมหนังสือที่ Resource Center สามารถลดขั้นตอนการยืมหนังสือทำให้ประหยัดเวลาในการยืมมากขึ้น
      2.3. คณะวิศวกรรมศาสตร์มีระบบยืม-คืนหนังสือใช้งานที่สะดวก และดีกว่ารูปแบบเดิม
      2.4. คณะวิศวกรรมศาสตร์มีแจ้งเตือนตรวจสอบหนังสือที่สะดวก และมีความเชื่อถือมากขึ้น
      2.5. คณะวิศวกรรมศาสตร์ลดภาระค่าใช้จ่ายในเรื่องกระดาษสำหรับจดยืม-คืนหนังสือ
    3. ผู้มีส่วนเสีย
      3.1. คณะวิศวกรรมศาสตร์ต้องสร้าง Database มารองรับฐานข้อมูลของหนังสือ และสถิติการยืม-คืนหนังสือ
      3.2. เจ้าหน้าที่คณะวิศวกรรมศาสตร์ที่เป็นผู้ดูแล และปรับปรุงระบบภายหลังจากระบบใช้งานจริง

  3. User Stories & Lo-Fi Mockups



    Feature: Add new book to book categories
      As an administrator
      I want to add new book
    Scenario: add new book to book categories
      Given I am on the “AddNewBookPage”
      Then I should see field for enter book detail
      When I fill all book detail
      And I press “Done”
      Then I should see dialog “success full”
      And I should be on the “AddNewBookPage”



    Feature: Delete a book list
      As an admin
      So that I can delete a book list and update a book list in library
      I want to delete a book list from library database
    Scenario: Try to delete a book list
      Given I am on the delete book page
      Then I should see “Search for a book”
      When I fill in “Search Name of book” with “OOP”
      And I press “Search”
      Then I should see “Book List Table of OOP” on the delete book page
      When I click “Delete column” with “OOP in PHP”
      And I press “Delete”
      Then I should be on the delete book page
      And I should see message “OOP in PHP was deleted”



    Feature: Admin can do detail to borrowing book of student
    Scenario: Try to make detail to borrow book of student
      Give I am on the borrow page
      Then I should see detail of student to borrow book
      When I input ID book to borrow
      And I press to borrow
      Then I should to update name book to student detail of student to borrow book

    Feature: I can return a book
      As an administrator
      I want to return a book
      And check charge price in case of overdue
    Background:
      Given I am on the “borrow/return page”
      Then I should see “Borrowed Book Table”
      And I should see “Red Riding Hood And Wonderland” in table
      When I check a “Return_Checkbox” in row of “Red Riding Hood And Wonderland”
      And I press “Return” I should not see “Red Riding Hood And Wonderland” in table
    Scenario: return a book without charge
      I should see message “Book Returns Success” over the “Borrowed Book Table”
    Scenario: return a book with charge
      I should see message “Charge Price is 30 baths. Thank you.” over the “Borrowed Book Table”



    Feature: Admin can Change a Charge Price or Checkout Allow For Each Category User
      As an admin
      So that I can change a price charge for each category user
      I want to edit a price charge or checkout allow then update those object
    Scenario: Try to change a price charge or checkout allow
      Given I am on the ChargePrice page
      Then I should see “price charge Table”
      When I edit “Bachelor’s Degree row” and “Price Charge column” with “5”
      And I press “update”
      Then I should be on the ChargePrice page
      And I should see “Bachelor’s Degree row” and “Price Charge column” update to “5”
